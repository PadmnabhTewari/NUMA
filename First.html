NUMA-Aware Memory Allocator

A Project by Devesh Mirchandani, Kartik Sharma, & Padmnabh Tewari

The Hidden Bottleneck in Multi-Socket Servers

In Non-Uniform Memory Access (NUMA) systems, CPUs can access their own local memory much faster than memory local to other CPUs. Standard memory allocation can inadvertently place data on a "remote" node, creating a significant performance chasm that slows down applications.

Our Solution: The Smart Interceptor

We've engineered a drop-in replacement for the standard `malloc()` function. By using the `LD_PRELOAD` mechanism, our shared library transparently intercepts memory requests and intelligently allocates memory on the correct NUMA node, eliminating remote access penalties without requiring any changes to the application's source code.

From Call to Allocation: The Four-Step Process

Step 1

Application calls standard `malloc()`.

⭣
Step 2

`LD_PRELOAD` intercepts the call and redirects it to our library.

⭣
Step 3

Our code identifies the current CPU's local NUMA node.

⭣
Step 4

Memory is allocated directly on the correct local node.

Projected Performance Gains

Our primary goal is a significant reduction in execution time for memory-intensive workloads. We are targeting a minimum 20% improvement over the standard allocator.

Maximizing Local Hits

Success is measured by tracking memory access patterns. Our allocator is designed to drastically increase `numa_hit` events (fast, local access) while minimizing slow `numa_miss` events.
